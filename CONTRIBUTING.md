# Contributing to Special Agent Ops

Thanks for your interest. This repo is a community resource and contributions are welcome.

## What we're looking for

- **New workflow examples** — real tasks you've run through the mission flow, with results
- **Improved prompts** — better instructions for any of the agent roles
- **Script improvements** — more robust secret patterns, better output formatting
- **Docs clarifications** — anywhere the guidance is unclear or missing edge cases
- **New templates** — for agent roles or use cases not yet covered
- **Honest failure reports** — what went wrong when you followed this workflow, and what you changed

## What we're not looking for

- Hype or marketing language
- Claims that agents are safe, reliable, or ready to work unsupervised
- Integrations that remove human review gates
- Contributions that treat AI as a replacement for developer judgment

## How to contribute

1. Fork the repo
2. Create a branch: `git checkout -b your-change-description`
3. Make your changes
4. Fill out the PR template (see `templates/PR_CHECKLIST.md` for the spirit of what we check)
5. Open a pull request with a clear description of what changed and why

## Standards

- Write in plain English. No jargon that a junior developer wouldn't know.
- Be specific. "Agents can expose secrets" is more useful than "agents have risks."
- Keep tone practical and honest. Slightly energetic is fine. Breathless hype is not.
- Scripts must be dependency-free or clearly document their dependencies.
- Prompt files should include a brief note on what AI tool and context they were tested with.

## Issues

Open an issue before starting large changes. It avoids duplicated effort and lets maintainers flag any direction concerns early.

## Code of conduct

Be straightforward. Be constructive. Don't be condescending about what tools people are using or where they are in their AI-assisted development journey.
